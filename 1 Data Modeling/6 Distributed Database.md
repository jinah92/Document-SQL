# 데이터 모델과 성능 - 분산 데이터베이스



## 분산 데이터베이스

---

- 물리적으로 떨어진 데이터베이스에 네트워크로 연결, 단일 데이터 베이스 이미지를 보여 주고 분산된 작업 처리를 수행
- 투명성 제공 (분할, 위치, 지역사상, 중복, 장애 및 병행)

### 분산 데이터베이스의 투명성

- 분할 투명성 : 고객은 각 단편의 사본이 여러 시스템에 저장됨을 인식하지 않음
- 위치 투명성 : 고객이 사용하려는 데이터의 저장 장소를 명시하지 않아도 됨
- 지역 사상 투명성 : 각 지역 시스템 이름과 무관한 이름이 사용 가능
- 중복 투명성 : 데이터베이스 객체가 여러 시스템에 중복되어 존재해도, 데이터의 일관성이 유지
- 장애 투명성 : 각 지역의 시스템, 통신망에 이상이 발생해도 무결성을 보장
- 병행 투명성 : 동시에 다수 트랜잭션이 발생해도 결과에 이상이 없음

### 설계 방식

- 상향식 설계 : 지역 스키마 ⇒ 전역 스키마
- 하향식 설계 : 전역 스키마 ⇒ 지역 스키마
- 기업에 여러 종류의 데이터베이스 관리 시스템이 있는 경우, 이기종 데이터베이스 관리 시스템으로 연동 ⇒ 데이터베이스 미들웨어 (ODBC, JDBC)

### 장, 단점

- 장점
    1. 높은 신뢰성, 가용성
    2. 병렬 처리 ⇒ 빠른 응답
    3. 시스템 용량 확장이 쉬움

- 단점
    1. 관리 및 통제가 어려움
    2. 보안관리가 어려움
    3. 데이터 무결성 관리 어려움
    4. 복잡한 설계
